# Runbook – Incident Response (DSGVO)

Ziel
- Datenpannen/Sicherheitsvorfälle strukturiert behandeln (Art. 33/34 DSGVO), Schäden minimieren.

Rollen & Kontaktkette
- Incident Lead; Privacy Officer/DPO; Security Eng.; Comms. On‑Call‑Liste pflegen.

Ablauf (Kurz)
1) Erkennen: Alert/Report triagieren; Schwere einstufen; Ticket eröffnen.
2) Eindämmen: Zugriff sperren, Keys rotieren, Systeme isolieren.
3) Analysieren: Umfang/Betroffene/Datenkategorien; Timeline/Root Cause.
4) Melden: Aufsichtsbehörde ≤ 72h (falls meldepflichtig); Betroffene informieren bei hohem Risiko.
5) Beheben: Patches/Hardening; Backfills (falls nötig); Monitoring schärfen.
6) Post‑Mortem: Ursachen, Maßnahmen, Lessons Learned; Fristen/Nachweise archivieren.

Evidenzen
- Log‑Auszüge (redacted), Change‑/Deploy‑Historie, Zugriffsnachweise, Kommunikationsvorlagen.

Hinweise
- Minimalprinzip in Kommunikation; Rechtsabteilung einbinden; DSAR‑Folgen beachten.
